# WorldTime 回归测试清单

每次完成一个优化阶段后，按照此清单逐项验证。
启动方式：`npx http-server . -p 8080`，浏览器访问 `http://localhost:8080`。

## 1. 世界地图 TAB (`#mapPanel`)
- [ ] 默认进入此 TAB，地图正常加载
- [ ] 6 个默认城市点位显示在正确经纬度
- [ ] 城市标签显示当地时间
- [ ] 切换投影按钮（Natural Earth ↔ Braun）有平滑过渡动画
- [ ] 切换晨昏线按钮（简单过渡 ↔ 自然过渡）正确切换
- [ ] 双击地图任意位置，地图旋转使该点居中
- [ ] 右下角按钮"城市列表"打开模态框
- [ ] 右上角"+"按钮打开城市选择模态框
- [ ] 太阳子午点和月亮子月点位置随时间移动
- [ ] 国际日期变更线和子夜线正确显示

## 2. 时间轴 TAB (`#timelinePanel`)
- [ ] 显示所有已选城市的水平 24h × 5天 色带
- [ ] 当前时间用红色游标线标记
- [ ] 协作窗口顶部显示"全员/部分/无重叠"标签
- [ ] 置顶城市图标（★）正确高亮
- [ ] 点击星标可置顶/取消置顶
- [ ] 农历日期、节气标签正确显示

## 3. 对照表 TAB (`#compPanel`)
- [ ] 矩阵表格显示所有已选城市 × 24 小时
- [ ] 每格颜色根据当地小时（白天/夜晚/晨昏）变化
- [ ] 当前时间列高亮
- [ ] 文本颜色与背景色形成足够对比度

## 4. Clock TAB (`#clockPanel`)
- [ ] 24 小时表盘正常渲染（12 在顶部，0 在底部）
- [ ] 时针、分针、秒针实时走动
- [ ] 太阳和月亮图标在表盘上显示当前位置
- [ ] 月相图标显示正确相位
- [ ] 晨昏线（日出/日落）颜色渐变正确
- [ ] **置顶城市的时针正确显示**（北京、纽约等）
- [ ] **城市标注沿时针方向竖排，3 个置顶城市颜色由深到浅**
- [ ] 5 分钟无操作后秒针自动隐藏，刷新频率降为每分钟
- [ ] 双击表盘重新启用秒针模式

## 5. 跨 TAB 一致性
- [ ] 在时间轴 TAB 修改置顶城市，切换到 Clock TAB 后时针同步变化
- [ ] 模态框中修改已选城市，地图/时间轴/对照表都同步更新
- [ ] 旋转地图后，重新加载页面状态保留

## 6. 持久化
- [ ] 关闭浏览器再打开，置顶/已选城市/地图状态都保留
- [ ] localStorage 中存在 `selectedCityIds` / `pinnedCityIds` / `mapState`

## 7. 主题
- [ ] 系统切换深色/浅色模式时，Clock TAB 表盘颜色自动适配

## 8. 壁纸模式
- [ ] 访问 `?wallpaper=1` URL 后，进入全屏壁纸模式
- [ ] 显示左上角时间和日期
- [ ] 通过另一个标签页修改设置，壁纸模式自动同步

## 9. 控制台无错误
- [ ] 打开 DevTools Console，无 `Uncaught Error` 红色报错
- [ ] 仅有的 `console.warn` 是预期的兜底（如时区数据不可用）

## 10. 性能基线
- [ ] 切换 TAB 响应迅速（<300ms）
- [ ] 待机 5 分钟后 CPU 占用降低（秒针模式停止）
